alpmbuild

the sane packaging format for alpm packages

specfiles

say goodbye to the old and primitive PKGBUILDs you’re used to, and say hello to the specfiles. packaging is easier than ever with specfiles. with simpler syntax and useful directives, you’ll find using specfiles to be a breeze.

speed

alpmbuild is programmed in golang, while makepkg is a shell script. this means your package spends more time actually building and less time parsing the build recipe.

error messages

we’re all humans, and we all make mistakes. alpmbuild is created with world-class error messages that tell you exactly what’s wrong with your specfile and help you fix it.

cross-distro format

ever wished you could target other distros like Debian and Fedora with a PKGBUILD? alpmbuild uses a specfile format compatible with rpmbuild and debbuild, allowing you to target many distros with the same specfile.

safety

alpmbuild by default is like rust when it comes to safety. you need to declare what files your package provides in the specfile by default, or opt into the unsafe mode. this ensures you know exactly what’s going to be put into your package before you build it.
